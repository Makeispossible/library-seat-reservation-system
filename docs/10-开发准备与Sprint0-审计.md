## D. 建议优化项

### 建议 1：统一"迁移"用词为"建库"

受影响的 3 处：

| 文件 | 当前表述 | 建议改为 |
|------|---------|---------|
| `README.md` 快速启动 | 自动执行迁移 + 种子数据 | 自动建库 + 种子数据 |
| `docs/10-开发准备与Sprint0.md` T0-08 | 首次 EF Core 迁移 + 建库建表 | 首次 EF Core 建库建表 |
| `docs/10-开发准备与Sprint0.md` T0-08 验收标准 | 数据库中生成了 3 张表 | 保持不变（这段没问题） |

理由：`docs/10` 第 7 节补足项已明确使用 `EnsureCreated()` + `DbInitializer`，不使用 EF Core 迁移工具。README 和 T0-08 中的"迁移"用词会造成读者困惑（读者会以为需要执行 `dotnet ef migrations add`）。

### 建议 2：将 T0-10 标记为已完成

T0-10（Git 初始化 + 首次提交 + 推送远端）已在当前轮次实际执行，应将任务板中 T0-10 的状态从"⏳ 待开发"更新为"✔️ 已完成"，并将"本轮是否验证通过"改为"✅ 是"。

### 建议 3：README 目录树中补充 `10-开发准备与Sprint0-审计.md`（可选）

如果本轮审计报告也会推送到仓库，建议在 README 的"当前已存在"目录树中补充本文件条目。
