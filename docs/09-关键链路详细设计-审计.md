## D. 建议优化项

### 建议 1：POST Create 增加座位存在性二次检查

在 `ReservationController.Create(POST)` 的开头增加：

```csharp
// 二次检查：座位在提交时是否仍存在且空闲
var seat = _db.Seats.FirstOrDefault(s => s.Id == id);
if (seat == null) return NotFound();

var todayReservations = _db.Reservations
    .Where(r => r.SeatId == id && r.Date == today && r.Status == "Pending")
    .ToList();
var currentStatus = StatusHelper.GetSeatDisplayStatus(seat.Status, todayReservations);
if (currentStatus != "空闲")
{
    ModelState.AddModelError("", "该座位当前不可预约");
    ViewBag.Seat = seat;
    SetUserContext();
    return View();
}
```

### 建议 2：明确代码片段中的 `query` 来源

在代码片段前补充一行注释：

```csharp
// 构建查询
var query = _db.Seats.AsQueryable();
if (!string.IsNullOrEmpty(area)) query = query.Where(s => s.Area == area);
if (!string.IsNullOrEmpty(floor)) query = query.Where(s => s.Floor == floor);
query = query.OrderBy(s => s.SeatNumber);
```

### 建议 3：将已决策项移出"待确认"表

将第 5.3 节中的第三行移出，改为在第 5.3 节末尾加注：

```
已决策：Seat.CreatedAt 不追加，按 docs/08 执行。
```
