## 2. 视觉方向

采用 **安静学习感 + 校园服务感** 主方向，对应 `docs/05-页面卡与UI规范.md` 中定义的视觉规范：

| 维度 | 采用方案 |
|------|---------|
| 主色 | `#2563EB` Blue-600（导航栏、按钮、链接） |
| 卡片背景 | `#FFFFFF`，圆角 12px，微阴影 |
| 页面背景 | `#F8FAFC` Slate-50 |
| 用户端导航 | 固定顶部，蓝色底 `#2563EB`，白色文字，56px 高 |
| 管理端导航 | 固定顶部，深灰蓝底 `#1E293B`，白色文字，56px 高 |
| 状态标签 | 圆角 badge，每种状态独立配色（文字+颜色双重标识） |
| 按钮 | 8px 圆角，主色/成功色/危险色/次要四种 |

### 禁用风格遵守情况

- ✅ 无暗黑/深色模式
- ✅ 无玻璃拟态
- ✅ 无渐变背景或大幅插图
- ✅ 无动效/转场动画
- ✅ 无自定义字体图标（使用 Bootstrap Icons 替代方案）
- ✅ 无大面积品牌色

## 6. 未接入逻辑

| 内容 | 原因 |
|------|------|
| 真实数据库查询 | 静态原型，无后端 |
| 账号切换持久化 | 使用 JS 前端模拟切换 |
| 预约冲突校验 | 前端模拟（显示错误提示） |
| 时间规则校验 | 需后端验证 |
| Session 守卫 | 需后端支持 |
| 真实登录验证 | 前端 JS 模拟（admin/admin123） |
| 动态状态计算 | 需后端 StatusHelper |
| 取消预约写入数据库 | 前端 JS 模拟状态变更 |
