## C. 必改项

### 必改 1：P0 功能计数错误

**位置**：第 7 节"保留项"

**当前写法**：
```
- P0 全部功能（9 项）
- P1 全部功能（8 项）
```

**问题**：矩阵中 P0 实际有 11 项：B1、B2、B3、A1、A4、A5、A6、D3、D4、C1、C7。

**修改建议**：
```
- P0 全部功能（11 项）
- P1 全部功能（8 项）
```

### 必改 2：Bootstrap 响应式不应为 P2

**位置**：第 4 节优先级矩阵，E1

**问题**：立项单规定技术栈包含 Bootstrap，且"Bootstrap 响应式布局，支持手机和电脑浏览"是写进立项单的要求。把框架自带的能力标为"有时间再做"会造成误导。

**修改建议**：将 E1 从 P2 提升为 **P1**，理由是"Bootstrap 是项目的 CSS 基础框架，使用即自带响应式，不需要额外开发"。同时更新 MVP 范围中"公共"部分的对应描述。

## D. 建议优化项

### 建议 1：取消"操作结果提示页"独立页面

建议在第 5 节 MVP 范围中，将页面 #9 从独立页面改为"在现有页面上集成操作结果提示（使用 TempData/Bootstrap Alert）"。PRD Lite 中真正需要的是 8 个页面 + 1 个通用消息机制。

### 建议 2：导航栏提升为 P1

没有导航栏，用户在页面之间跳转只能靠手动输入 URL 或浏览器"后退"按钮。对于课堂演示而言不可接受。建议将 E2 提升为 P1，至少实现简单的顶部导航链接。

### 建议 3：明确"区域/楼层筛选"的交互方式

建议在 PRD Lite 中决策：下拉框 or 标签页 or 侧栏筛选。推荐用下拉框（实现简单，一个 SelectList 即可）。

### 建议 4：第 5 节的 MVP 共约页面数量

如果采纳"操作结果提示页"不独立成页、导航栏提升为 P1，"操作结果提示"合并到现有页面，实际页面数变为 **8 个**。建议核实后更新页面数字。
