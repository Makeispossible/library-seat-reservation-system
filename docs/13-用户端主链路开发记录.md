## 4. 运行时问题修复

### 4.1 HomeController SwitchUser 参数类型

| 项目 | 内容 |
|------|------|
| **问题** | `SwitchUser(int userId)` 使用非可空 `int`，当下拉框 `value=""` 时模型绑定失败 |
| **风险** | 下拉框首次选中"切换账号"`value=""` 时，绑定 `int` 会抛 500 错误 |
| **修复** | 改为 `SwitchUser(int? userId)`，方法内 `if (userId.HasValue)` 判断 |
| **文件** | `src/LibrarySeatReservation.Web/Controllers/HomeController.cs` |
| **影响范围** | 仅 `SwitchUser` 方法签名和内部 null 判断 |
| **验证** | ✅ 修复后下拉框切换任意账号正常，空值不处置 |

### 4.2 其他未发现运行时问题

| 检查项 | 结果 |

| 404 路由 | 仅 `LibrarySeatReservation.Web.styles.css`（Razor Page 可选文件，不影响功能）和 `favicon.ico` |
| Session 空引用 | ✅ 所有 View 中 `ViewBag.CurrentUser` 使用了 `??` 兜底 |
| 数据库连接 | ✅ LocalDB 正常，种子数据 5 学生 + 12 座位已写入 |
| 筛选空状态 | ✅ 当 Model.Count == 0 时显示空状态卡片 |
