## C. 必改项

### 必改 1：管理端缺少未登录守卫逻辑

**位置**：第 3 节"管理端页面树"

**问题**：管理端页面树只画了"登录 → 预约管理 → 座位管理 → 统计"的登录后流程，但没有画出**未登录时直接访问管理页面的拦截逻辑**。

**修改建议**：在管理端页面树的最外层补充如下守卫：

```
访问 /Admin/* 任意页面
    │
    ├─ 已登录 → 正常渲染该页面
    │
    └─ 未登录 → 重定向到 /Admin/Login
```

这个守卫是 ASP.NET Core 中 `[Authorize]` 属性或自定义 `Session` 检查的标准做法。应在页面树中明确画出，否则设计师/开发者可能忘记实现。

## D. 建议优化项

### 建议 1：链路 C 补充详细步骤

当前链路 C 只有 2 行，与其余 4 条链路的详细程度不一致。建议改为：

```
链路 C：多账号对比验证

① 用户首页 → 切换学生A
② 座位列表 → 点击空闲座位 → 座位详情 → 预约提交 → 我的预约（确认已预约）
③ 切换学生B（从首页切换）
④ 座位列表 → 找到同一座位 → 座位详情
⑤ 确认该座位此时段标记为"已预约"（或不在空闲列表中）
```

### 建议 2：补充用户端导航栏结构

在用户首页的页面树中，将"链接"替换为导航栏结构描述，例如：

```
用户端导航栏（所有用户页面顶部可见）
┌────────────────────────────────────────┐
│  首页  │  座位列表  │  我的预约        │
│            当前用户：学生A  [切换▼]     │
└────────────────────────────────────────┘
```

### 建议 3：管理端"退出"跳转目标

建议在管理导航栏中注明退出跳转目标：

`[退出] → 清除 Session → 跳转管理员登录页`

### 建议 4：链路 E 增加返回步骤

链路 E（查看统计）当前只有"① 登录 → ② 查看统计"，建议补充第③步：

```
③ 点击导航 → 回到预约管理 / 座位管理
```

### 建议 5：预约提交页的日期字段

建议在预约提交页的字段表中明确日期字段的存在形式：

- 如果隐藏：说明"日期为隐藏字段，自动取服务器当天日期"
- 如果去掉：说明"本期仅支持预约当日，不显示日期选择"
