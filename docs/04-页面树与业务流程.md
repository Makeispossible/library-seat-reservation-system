# 页面树与业务流程 — 图书馆座位预约系统

> 本文档基于 `docs/03-PRD-Lite.md` 编写。

## 1. 页面总数与页面分布

### 总数：9 页

| 端 | 页面数 | 包含页面 |
|----|--------|----------|
| 用户端 | 5 页 | 用户首页、座位列表页、座位详情页、预约提交页、我的预约页 |
| 管理端 | 4 页 | 管理员登录页、预约管理页、座位管理页、统计页 |

## 2. 用户端页面树

```
用户首页 (/)
│
├── 体验账号切换（下拉框，页面顶部）
│   └── 选择学生A~E → 页面刷新 → 显示当前用户
│
├── [链接] 座位列表页
│   └── /Seats
│
└── [链接] 我的预约页
    └── /Reservation/My
```

### 座位列表页 (`/Seats`)

```
座位列表页
│
├── 区域/楼层筛选（两级下拉框）
│   └── 筛选结果 → 刷新座位卡片列表
│
├── [卡片: 座位A] → 点击 → 跳转座位详情页
├── [卡片: 座位B] → 点击 → 跳转座位详情页
└── [卡片: 座位N] → 点击 → 跳转座位详情页
```

### 座位详情页 (`/Seats/Detail/{id}`)

```
座位详情页
│
├── 座位信息展示（编号、区域、楼层、状态）
├── 今日预约时段列表
│   ├── 有时段 → 显示时段时间线
│   └── 无时段 → 显示"今日暂无预约记录"
│
└── [按钮: 预约此座位]（仅空闲状态可见）
    └── 点击 → 跳转预约提交页
```

### 预约提交页 (`/Reservation/Create/{seatId}`)

```
预约提交页
│
├── 表单: 座位号（只读）
├── 表单: 开始时间（下拉选择 时:分）
├── 表单: 结束时间（下拉选择 时:分）
├── 时间规则提示（最小30分钟，最长4小时）
│
├── [提交] → 校验
│   ├── 成功 → 跳转我的预约页 + 成功消息
│   └── 失败 → 当前页显示错误原因
│
└── [返回] → 回座位详情页
```

### 我的预约页 (`/Reservation/My`)

```
我的预约页
│
├── 预约记录列表（按时间倒序）
│   ├── 记录1: 座位号、区域、日期、时段、状态
│   │   └── [按钮: 取消预约]（仅"待开始"状态可见）
│   │       └── 点击 → Bootstrap 确认弹窗
│   │           ├── 确认 → 状态变已取消 → 列表刷新
│   │           └── 取消 → 关闭弹窗
│   ├── 记录2: ...
│   └── 记录N: ...
│
└── [链接] 继续预约 → 跳转座位列表页
```

## 4. 用户主链路

### 链路 A：完整预约流程（5 步闭环）

```
① 用户首页
   ├─ 切换体验账号
   └─ 点击"座位列表"
       │
       ▼
② 座位列表页
   ├─ 筛选区域/楼层
   └─ 点击空闲座位卡片
       │
       ▼
③ 座位详情页
   ├─ 查看座位信息 + 今日时段
   └─ 点击"预约此座位"
       │
       ▼
④ 预约提交页
   ├─ 选择开始时间 + 结束时间
   └─ 点击提交
       │
       ▼
⑤ 我的预约页（成功 → 看到新记录）
```

### 链路 B：取消预约流程（3 步闭环）

```
① 我的预约页
   └─ 找到"待开始"预约 → 点击"取消预约"
       │
       ▼
② 确认弹窗
   └─ 点击"确定"
       │
       ▼
③ 我的预约页（刷新 → 状态变"已取消"，取消按钮消失）
```

### 链路 C：多账号对比

```
① 用户首页 → 切换学生A → 预约座位
② 用户首页 → 切换学生B → 查看同一座位 → 时段已标记（非空闲）
```

## 6. 状态流转

### 座位可见状态

> 座位状态采用动态计算方式，不在数据库中持久化流转。数据库 seat.Status 仅存储"空闲"和"维护中"两个持久状态，"已预约""使用中"由系统根据关联的预约记录实时计算。

```
┌──────────┐      预约成功      ┌──────────┐
│          │ ────────────────→ │          │
│  空闲    │                    │  已预约  │
│  (db)    │ ←──────────────── │  (动态)  │
│          │    取消预约        │          │
└────┬─────┘                    └──────────┘
     │                              │
     │ 管理员标记     当前时间≥开始时间
     ▼                              ▼
┌──────────┐               ┌──────────┐
│  维护中   │               │  使用中   │
│  (db)    │               │  (动态)   │
└──────────┘               └──────────┘
                                  │
                            当前时间≥结束时间
                                  │
                                  ▼
                            自动视为空闲
                            (动态展示)
```

### 预约状态

```
┌──────────┐      开始时间到达     ┌──────────┐
│          │ ──────────────────→ │          │
│  待开始   │     （动态计算）      │  使用中   │
│  (db)    │                      │  (动态)   │
└────┬─────┘                      └────┬─────┘
     │                                  │
     │ 用户取消                   结束时间到达
     ▼                          （动态计算）
┌──────────┐                          │
│  已取消   │                          ▼
│  (db)    │                  ┌──────────┐
└──────────┘                  │  已完成   │
                              │  (动态)   │
                              └──────────┘
```

**状态存储说明：**

| 字段 | 存储方式 |
|------|----------|
| 座位状态（空闲/维护中） | 数据库 seat.Status 字段持久化 |
| 座位状态（已预约/使用中） | 动态计算，不存库 |
| 预约状态（待开始/已取消） | 数据库 reservation.Status 字段持久化 |
| 预约状态（使用中/已完成） | 动态计算，不存库 |

**动态计算规则：**

```
当前时间 < 预约开始时间 → 显示"待开始"
开始时间 ≤ 当前时间 < 结束时间 → 显示"使用中"
当前时间 ≥ 结束时间 → 显示"已完成"
```

## 7. 页面与流程关系说明

### 页面 → 流程映射

| 页面 | 所属链路 | 角色 |
|------|----------|------|
| 用户首页 | A-①, C | 入口，切换身份 |
| 座位列表页 | A-② | 浏览+筛选座位 |
| 座位详情页 | A-③ | 查看详情，触发预约 |
| 预约提交页 | A-④ | 核心业务操作 |
| 我的预约页 | A-⑤, B | 结果查看+取消操作 |
| 管理员登录页 | D-① | 管理入口 |
| 预约管理页 | D-② | 查看预约全景 |
| 座位管理页 | D-③ | 管理座位 |
| 统计页 | E | 数据概览 |

### 页面间跳转总结

```
用户首页 ──→ 座位列表页 ──→ 座位详情页 ──→ 预约提交页
    │                                              │
    └──────────────→ 我的预约页 ←───────────────────┘
                          │
                          └── 取消操作（当前页弹窗，不跳转）

管理员登录页 ──→ 预约管理页 ──→ 座位管理页
                     │              │
                     └──→ 统计页 ←──┘
```

## 8. 当前范围内页面

| # | 页面 | 路由 | 所属端 | 核心操作 |
|---|------|------|--------|----------|
| 1 | 用户首页 | `/` | 用户端 | 切换体验账号，导航到其他页 |
| 2 | 座位列表页 | `/Seats` | 用户端 | 查看座位，区域筛选 |
| 3 | 座位详情页 | `/Seats/Detail/{id}` | 用户端 | 查看详情，触发预约 |
| 4 | 预约提交页 | `/Reservation/Create/{seatId}` | 用户端 | 选时提交预约 |
| 5 | 我的预约页 | `/Reservation/My` | 用户端 | 查看+取消预约 |
| 6 | 管理员登录页 | `/Admin/Login` | 管理端 | 账号密码登录 |
| 7 | 预约管理页 | `/Admin/Reservations` | 管理端 | 查看全部预约 |
| 8 | 座位管理页 | `/Admin/Seats` | 管理端 | 查看+添加座位 |
| 9 | 统计页 | `/Admin/Stats` | 管理端 | 展示统计数据 |

### 与 PRD Lite 差异说明

| 项目 | PRD Lite | 本文件 | 原因 |
|------|----------|--------|------|
| 管理端页数 | 3 页（不含统计） | 4 页（含统计） | 固定口径要求加入统计页 |
| 统计页 | 暂缓 | 恢复纳入 | 固定口径要求 |
| 预约状态流转 | "自动"表述 | 动态计算方案 | 审计修正 |
