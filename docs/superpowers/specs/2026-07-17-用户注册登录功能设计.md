# 用户注册登录功能设计

## 1. 概述

为图书馆座位预约系统添加用户名密码注册登录功能，替代原有的体验账号切换模式。

## 2. 模型变更

### StudentUser 表扩展

| 现有字段 | 变更 |
|----------|------|
| Id (int, PK) | 不变 |
| Name (string, 50) | 不变 |
| **Username** (string, 50) | **新增，唯一索引** |
| **PasswordHash** (string, 200) | **新增** |
| Reservations (Navigation) | 不变 |

密码使用 `Microsoft.AspNetCore.Identity.PasswordHasher<StudentUser>` 进行 PBKDF2 哈希存储。

## 3. 新增文件

### Controllers/AccountController.cs

| 路由 | 方法 | 功能 |
|------|------|------|
| `GET /Account/Register` | `Register()` | 显示注册表单 |
| `POST /Account/Register` | `Register(RegisterViewModel)` | 校验 → 哈希密码 → 写入 DB → 自动登录 → 跳转首页 |
| `GET /Account/Login` | `Login()` | 显示登录表单 |
| `POST /Account/Login` | `Login(LoginViewModel)` | 校验凭证 → 设置 Session → 跳转首页 |
| `POST /Account/Logout` | `Logout()` | 清除 Session → 跳转首页 |

### ViewModels 新增

```csharp
public class RegisterViewModel
{
    [Required] public string Username { get; set; }  // 3-50 字符，字母数字或下划线
    [Required] public string Name { get; set; }      // 真实姓名，1-50 字符
    [Required][MinLength(6)] public string Password { get; set; }
    [Compare("Password")] public string ConfirmPassword { get; set; }
}

public class LoginViewModel
{
    [Required] public string Username { get; set; }
    [Required] public string Password { get; set; }
}
```

### Views/Account/Register.cshtml + Login.cshtml

Bootstrap 表单，与服务端校验结合，注册时 Username 唯一性由后端保证。

## 4. 删除/修改文件

### 删除

| 文件 | 原因 |
|------|------|
| `Controllers/HomeController.cs` → `SwitchUser` 动作 | 不再需要切换账号 |
| `Views/Shared/_Layout.cshtml` → 体验账号下拉框 | 不再需要 |
| `Views/Home/Index.cshtml` → 体验账号列表 | 不再需要 |

### 修改

| 文件 | 变更 |
|------|------|
| **Models/StudentUser.cs** | 添加 `Username`、`PasswordHash` |
| **Data/AppDbContext.cs** | 为 `StudentUser.Username` 添加唯一索引 |
| **Data/DbInitializer.cs** | 移除 5 个种子体验账号 |
| **Controllers/HomeController.cs** | `Index` 不再加载体验账号列表，展示登录状态 |
| **Views/Shared/_Layout.cshtml** | 导航栏右端替换为登录/注册/退出链接 |
| **Views/Home/Index.cshtml** | 简化首页，去掉体验账号卡片，改为欢迎页 |

### 登录状态

复用现有的 Session key:
- `UserId` (int) — 当前登录用户的 StudentUser.Id
- `UserName` (string) — 当前登录用户的 StudentUser.Name

所有现有使用 `HttpContext.Session.GetInt32("UserId")` 的代码无需改动（ReservationController、SeatsController 等）。

## 5. 数据迁移

```bash
dotnet ef migrations add AddUserAuth --project src/LibrarySeatReservation.Web
```

迁移包含：
- 新增 `StudentUsers.Username` 列（非空，唯一索引）
- 新增 `StudentUsers.PasswordHash` 列（可为空，兼容已有种子数据）
- 删除的体验账号无需迁移处理（在 DbInitializer 中移除种子数据即可）

## 6. 边界情况

| 场景 | 处理 |
|------|------|
| 注册时用户名已存在 | 返回错误"该用户名已被注册" |
| 登录时用户名不存在 | 返回错误"用户名或密码错误" |
| 登录时密码错误 | 同上（不暴露哪个字段错） |
| 未登录访问需登录页面 | 跳转登录页（现有各 Controller 已有 Session 守卫） |
| 密码强度 | 最少 6 位（由 ViewModel 校验） |
| 用户名格式 | 字母/数字/下划线，3-50 位 |

## 7. 不包含的功能（YAGNI）

- 邮箱验证
- 密码重置
- 角色/权限分级（仅 Admin 硬编码不变）
- 第三方 OAuth 登录
