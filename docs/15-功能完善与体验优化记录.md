## 2. 本轮执行内容

### 2.1 预约冲突校验 + 时间规则（T3-01）

**实现位置：** `ReservationController.Create(POST)`

- **冲突检测：** 提交预约时检查同一座位在同一时段是否已有 Pending 预约
- **时间规则：** 最短预约 30 分钟，最长 4 小时
- **实现方式：** 服务端显式检查，ModelState 无效则返回表单页并显示错误消息
- **校验逻辑：**
  - `StartTime >= 当前时间 + 30 分钟`（防止过去时间预约）
  - `EndTime - StartTime >= 30 分钟`（最短时长）
  - `EndTime - StartTime <= 4 小时`（最长时长）
  - 查询 Reservations 表：`SeatId == seatId && Date == today && Status == Pending && StartTime < endTime && EndTime > startTime`（时段重叠检测）

### 2.2 座位状态守卫（T3-02）

**实现位置：** `SeatsController.Detail()`、`ReservationController.Create(GET)`

- 座位状态为 `Maintenance` 时，预约按钮禁用或隐藏
- 座位详情页显示"该座位维护中，暂不可预约"提示
- 预约提交页检查座位状态，非 Available 时重定向回详情页

### 2.3 取消预约守卫（T3-03）

**实现位置：** `StatusHelper.CanCancel()`

- 仅 `Pending` 状态可取消
- 仅 `StartTime > 当前时间` 可取消（已开始的预约不可取消）
- 管理端取消不受时间限制（管理员可取消任何 Pending 预约）
- 取消操作使用 POST 方式，防止直接 URL 访问

### 2.4 状态显示统一（T3-04）

**实现位置：** `Helpers/StatusHelper.cs`

- 座位动态状态计算（Available / Reserved / InUse / Maintenance）
- 预约动态状态计算（Pending / InUse / Completed / Cancelled）
- 统一的状态标签样式（Badge 颜色对）
- 用户端和管理端使用同一套状态计算逻辑

### 2.5 座位状态切换（T3-05）

**实现位置：** `AdminController.ToggleSeatStatus()`

- 管理员可在座位管理页切换座位状态：Available ↔ Maintenance
- POST 请求，成功后重定向回座位管理页

### 2.6 管理员取消预约（T3-06）

**实现位置：** `AdminController.CancelReservation()`

- 管理员可取消任何状态的预约
- POST 请求，成功后重定向回预约管理页

### 2.7 统计页（T3-07）

**实现位置：** `AdminController.Stats()`、`Views/Admin/Stats.cshtml`

- 实时从数据库计算：
  - 座位总数：`Seats.Count()`
  - 今日预约数：`Reservations.Count(d => d.Date == today && d.Status != Cancelled)`
  - 使用率：`今日有预约的座位数 / 座位总数 × 100%`
  - 区域分布：`GROUP BY Area`
- 统计页渲染为数据卡片（StatsCard 样式）

### 2.8 空状态处理（T3-08）

| 页面 | 空状态处理 |
|------|-----------|
| 座位列表（筛选无结果） | 显示"该区域暂无座位" + 重置筛选按钮 |
| 座位详情（今日无预约） | 显示"今日暂无预约记录" |
| 我的预约（无预约） | 显示"你还没有预约记录" + "去预约"按钮 |
| 预约管理（无记录） | 表格显示"暂无数据" |
| 座位管理（无座位） | 显示"暂无座位，请添加" + 添加按钮 |
| 统计页（无数据） | 使用率显示 0%，其他数字正常 |

### 2.9 提示信息完善（T3-09）

- 使用 TempData + Bootstrap Alert 实现操作结果提示：
  - 预约成功 → 绿色提示
  - 取消成功 → 绿色提示
  - 操作失败 → 红色提示
  - 校验错误 → 表单内红色文字
