# PRD Lite — 图书馆座位预约系统

> 本文档基于 `docs/01-项目立项单.md` 和 `docs/02-需求分析与MVP确认.md` 编写。

---

## 1. 项目概述

图书馆座位预约系统是一个轻量级的 Web 应用，让学生可以通过浏览器在线查看座位空闲情况、预约座位、取消预约；管理员可以在后台管理座位和查看预约记录。系统采用体验账号切换方式，无需注册即可使用。

| 项目 | 内容 |
|------|------|
| 项目名称 | 图书馆座位预约系统 |
| 技术栈 | ASP.NET Core MVC (.NET 9) + Razor + SQL Server LocalDB + EF Core + Bootstrap |
| 前端方案 | Razor 服务端渲染 + Bootstrap 响应式布局 |
| 数据库 | SQL Server LocalDB（优先）/ SQLite（降级兜底） |
| 交付方式 | GitHub / Gitee 仓库完整源码 |

---

## 2. 项目目标

1. 学生可查座位、约座位、取消预约，无需注册。
2. 管理员可管理座位、查看预约记录。
3. 项目源码可运行、可演示、可提交为课程作业。

## 4. 使用场景

### 场景一：高峰期找座位
期末周图书馆爆满，学生用手机打开系统，看到 3 楼有 2 个空闲座位，预约了一个 2 小时时段，直接到座位坐下。

### 场景二：预约后取消
学生预约了下午的座位但临时有事，在"我的预约"页面点击取消，座位状态立即释放。

### 场景三：管理员加座位
图书馆新添了一批桌椅，管理员登录后台 → 座位管理 → 添加座位，输入区域、楼层、编号，前台立刻可见。

## 5. 用户端核心功能

### F1 — 体验账号切换

| 项目 | 说明 |
|------|------|
| 入口 | 首页顶部下拉框 |
| 可用账号 | 学生A ~ 学生E（预设 5 个） |
| 切换后 | 首页顶部显示"当前用户：学生X" |
| 持久化 | 存储在 Session 中，刷新页面不丢失 |
| **验收标准** | 选中一个账号后页面刷新，顶部显示该用户名。切换不同账号，"我的预约"显示不同数据 |

### F2 — 查看座位列表

| 项目 | 说明 |
|------|------|
| 展示方式 | 卡片网格，每张卡片显示座位号 + 区域 + 当前状态 |
| 状态颜色 | 空闲（绿色）、已预约（橙色）、使用中（蓝色）、维护中（灰色） |
| 筛选 | 按区域 + 楼层，两级下拉框筛选 |
| **验收标准** | 页面加载后显示 ≥10 个座位。选择区域"3楼"后列表只显示 3 楼座位 |

### F3 — 查看座位详情

| 项目 | 说明 |
|------|------|
| 展示内容 | 座位编号、区域、楼层、当前状态 |
| 时段面板 | 今日已有时段列表（如果今日无预约，显示"今日暂无预约记录"） |
| 操作入口 | "预约此座位"按钮（仅空闲状态可点击） |
| **验收标准** | 点击一个空闲座位进入详情页，能看到座位信息和"预约此座位"按钮 |

### F4 — 提交预约

| 项目 | 说明 |
|------|------|
| 入口 | 座位详情页 → 点击"预约此座位" |
| 表单字段 | 日期（自动为今天）、开始时间、结束时间 |
| 时间规则 | 最小 30 分钟，最长 4 小时，不可跨天，开始时间 < 结束时间 |
| 冲突校验 | 同一座位同一时间区间不能有重叠的有效预约 |
| 预约成功 | 页面跳转至"我的预约"并显示成功消息 |
| 预约失败 | 当前页显示错误原因（"该时段已被预约"等） |
| **验收标准** | 选择一个空闲座位和未来时段，提交后在我的预约中可以看到该记录。选择同一座位同一时段再约，提示冲突 |

### F5 — 我的预约

| 项目 | 说明 |
|------|------|
| 展示内容 | 当前用户的所有预约记录，每条显示：座位号、区域、日期、时段、状态 |
| 预约状态 | 待开始 / 使用中 / 已完成 / 已取消 |
| 操作 | 对"待开始"状态的预约，提供"取消预约"按钮 |
| **验收标准** | 当前用户能看到自己预约的记录，记录显示正确的座位和时段 |

### F6 — 取消预约

| 项目 | 说明 |
|------|------|
| 入口 | "我的预约"页 → 点击"取消预约"按钮 |
| 确认 | Bootstrap 弹窗确认："确定取消 [座位号] 的预约吗？" |
| 结果 | 预约状态变更为"已取消"，座位释放，页面显示取消成功消息 |
| 限制 | 仅"待开始"状态的预约可取消 |
| **验收标准** | 对一个待开始的预约点击取消，确认弹窗出现，确认后状态变为"已取消"。已取消的预约不再显示取消按钮 |

## 8. 核心业务流程

### 流程一：学生预约座位

```
首页（切换账号）
  → 座位列表（按区域筛选）
    → 点击座位 → 座位详情
      → 点击"预约此座位" → 预约表单（选时间）
        → 提交 → 系统校验
          ├─ 成功 → 跳转到"我的预约" → 显示预约成功消息
          └─ 失败 → 当前页显示错误原因
```

### 流程二：取消预约

```
我的预约
  → 点击待开始预约的"取消预约"按钮
    → 弹窗确认
      ├─ 确认 → 状态变更为已取消 → 列表刷新
      └─ 取消 → 关闭弹窗
```

### 流程三：管理员管理

```
管理员登录页（输入账号密码）
  → 座位管理（查看所有座位 / 添加新座位）
  → 预约记录（查看全部预约，按时间倒序）
```

## 10. 验收标准

### 全系统

1. 项目通过 `dotnet run` 能正常启动，浏览器访问 `http://localhost:xxxx` 看到首页。
2. 首页加载后能看到体验账号切换下拉框，选中一个账号后顶部显示"当前用户：学生X"。
3. 导航栏包含"座位列表""我的预约"链接（用户端）和管理员登录入口。

### 用户端

4. 座位列表显示 ≥10 个座位，空闲座位为绿色标识。
5. 选择区域筛选后，列表只显示该区域的座位。

## 12. 状态流转说明

### 座位状态

```
维护中 ←→ 空闲 → 已预约 → 使用中 → 空闲
```

- 座位默认创建时为"空闲"。
- 学生成功预约后 → 变为"已预约"。
- 预约开始时间到达 → 自动变为"使用中"（或管理员手动标记）。
- 预约结束时间到达 → 自动变为"空闲"。
- 取消预约 → 立即变为"空闲"。
- 管理员可在后台将座位设为"维护中"，恢复后回到"空闲"（本期通过数据库直接修改）。

### 预约状态

```
待开始 → 使用中 → 已完成
   ↓ (取消)
 已取消
```

- 提交预约成功 → "待开始"。
- 到达开始时间 → "使用中"。
- 到达结束时间 → "已完成"。
- 用户在"待开始"状态点击取消 → "已取消"。
- 已取消的预约不再影响座位空闲判断。

## 14. 暂不开发内容

### 立项单继承的不做项

- 支付功能
- 扫码签到
- 消息推送
- 复杂账号体系
- 多馆区联动

|------|----------|
| 统计页 | 数据量小，演示用数据库查询截图代替 |
| 编辑/删除座位 | MVP 只做"添加"，编辑删除用数据库工具 |

| 404/500 自定义错误页 | ASP.NET Core 默认页够用 |
| 空状态提示 | 种子数据保证运行后不空 |

## 15. 风险与假设

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 时间不够做不完 | 无法完整演示 | 优先完成用户端 P0，管理端只做登录+预约记录 |
| Student 页面数量失控 | 项目延期 | 严格按 8 页执行，不临时加页 |
| LocalDB 装不上 | 项目无法运行 | 降级 SQLite，连接字符串改一行即可 |
| EF Core 迁移报错 | 演示数据丢失 | 每次改模型后执行 `dotnet ef migrations add` |
| AI 生成代码不稳定 | 功能不可用 | AI 代码必须走读 + 手动测试 |
| 界面太丑 | 演示效果差 | 使用 Bootstrap 自带组件，不写自定义 CSS |
