# 项目立项单 — 图书馆座位预约系统

## 1. 项目基本信息

| 项目 | 内容 |
|------|------|
| 项目名称 | 图书馆座位预约系统 |
| 英文名 | Library Seat Reservation System |
| 技术栈 | ASP.NET Core MVC (.NET 9) + Razor + SQL Server LocalDB + EF Core + Bootstrap |
| 前端方案 | Razor 服务端渲染 + Bootstrap 响应式布局 |
| 数据库 | SQL Server LocalDB（本地文件数据库，无需安装独立数据库服务） |
| 交付方式 | GitHub / Gitee 仓库完整源码 |
| 开发工具 | Visual Studio 2022+ / VS Code + dotnet CLI |

## 2. 项目背景

高校图书馆普遍存在"座位被占但人不在""高峰期一座难求"的问题。目前的常见做法是学生用书本或水杯占座，既不公平也无法流转。本项目旨在做一个轻量的在线座位预约系统，让学生在手机上就能查看座位、预约座位、取消预约，管理员也能在后台管理座位和查看使用情况。

这是课程第三阶段的实战项目，模拟入职后独立负责第一个小项目的完整流程：从立项、需求分析、设计、编码到交付。

## 3. 目标用户

- **学生用户（主要用户）**：在校大学生，使用浏览器查看和预约座位，需要体验账号即可使用，无需注册流程。
- **管理员（图书馆工作人员）**：管理座位（增删改）、查看预约记录、查看统计数据。

## 4. 项目目标

1. 实现一个可通过浏览器访问的座位预约 Web 应用。
2. 学生用户可以查看座位空闲情况、预约座位、取消预约。
3. 管理员可以管理座位和预约。
4. 项目代码可运行、可演示、可提交作为课程作业。

## 5. 本期必须完成内容

### 用户端

- 用户首页 —— 展示系统名称、功能入口
- 座位列表页 —— 按区域/楼层展示所有座位及其状态（空闲/已预约/使用中）
- 座位详情页 —— 查看单个座位的信息和今日预约时段
- 预约提交 —— 选择座位、选择时间段、提交预约
- 我的预约 —— 查看当前用户的所有预约记录
- 取消预约 —— 对未开始的预约执行取消操作
- 轻量模拟登录 —— 切换体验账号（如"学生A""学生B"），不做真实注册

### 管理端

- 管理员登录 —— 使用 Session / Cookie 登录
- 座位管理 —— 添加、编辑、删除座位（含区域、楼层、座位编号）
- 预约管理 —— 查看所有预约、手动结束预约
- 统计页 —— 展示座位使用率、热门时段等基础图表

### 公共

- 使用 EF Core 完成数据库迁移和初始化种子数据
- Bootstrap 响应式布局，支持手机和电脑浏览

## 6. 本期暂不开发内容

> 以下内容已确认本期不包含，设计时不要预留接口，不要做扩展考虑。

- ❌ 支付功能（预约免费）
- ❌ 扫码签到（图书馆门口放二维码扫码入场）
- ❌ 消息推送（预约成功短信/邮件通知）
- ❌ 复杂账号体系（统一认证、微信登录、学生证绑定）
- ❌ 多馆区联动（本期仅单馆区）
- ❌ 违规扣分/信用分机制
- ❌ 座位热力图或三维地图
- ❌ 移动端 App（仅 Web 端）

## 7. 页面规模与页面分布

共约 **10 个页面**。

### 用户端（6 页）

| 页面 | 路由 | 说明 |
|------|------|------|
| 用户首页 | `/` | 欢迎页 + 功能入口 |
| 座位列表 | `/Seats` | 按区域/楼层展示座位卡片 |
| 座位详情 | `/Seats/Detail/{id}` | 座位信息 + 今日预约时间线 |
| 预约提交 | `/Reservation/Create/{id}` | 选择时段 + 确认预约 |
| 我的预约 | `/Reservation/My` | 当前用户的所有预约 |
| 取消预约 | `/Reservation/Cancel/{id}` | 确认取消（操作后跳回列表） |

### 管理端（4 页）

| 页面 | 路由 | 说明 |
|------|------|------|
| 管理员登录 | `/Admin/Login` | 账号密码登录 |
| 座位管理 | `/Admin/Seats` | 座位 CRUD 列表 |
| 预约管理 | `/Admin/Reservations` | 全部预约记录管理 |
| 统计页 | `/Admin/Stats` | 座位使用统计 |

## 8. 关键业务主流程

### 流程一：学生预约座位

```
打开首页 → 查看座位列表 → 选择空闲座位 → 查看座位详情
→ 选择预约时间段 → 提交预约 → 系统校验（座位是否仍空闲、时间是否有效）
→ 预约成功 → 跳转到"我的预约"
```

### 流程二：管理员管理座位

```
管理员登录 → 进入管理后台 → 座位管理 → 添加/编辑/删除座位
→ 保存后列表刷新
```

### 流程三：管理员处理预约

```
管理员登录 → 进入预约管理 → 查看所有预约记录
→ 可手动结束已开始的预约 → 查看统计数据
```

## 9. 成功标准

> 以下标准均为可演示动作，验收时逐条检查。

1. 打开浏览器访问 `http://localhost:xxxx`，能看到系统首页。
2. 点击"座位列表"，能看到 10 个以上的座位数据（种子数据预置）。
3. 选择一个空闲座位，提交预约后，"我的预约"页面能看到这条记录。
4. 对未开始的预约点击"取消"，预约状态变更为"已取消"。
5. 管理员使用固定账号登录后台，能添加一个新座位，前台立刻可见。
6. 管理员后台能看到所有预约记录。
7. 统计页能显示座位的使用率数据（至少一个简单图表或数字统计）。
8. 在手机上用浏览器访问页面布局不乱。
9. 项目能通过 `dotnet run` 一键启动，无需额外配置数据库连接。

## 10. 风险与假设

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 时间不够，做不完所有页面 | 无法完整演示 | 优先做完用户端，管理端做最少可用版本 |
| 页面数量失控，越加越多 | 项目延期 | 严格按立项单的 10 个页面执行，不临时加页 |
| SQL Server LocalDB 学生机上装不上 | 项目无法运行 | 可降级为 SQLite 作为兜底方案（但优先用 LocalDB） |
| EF Core 迁移报错或数据丢失 | 演示数据出问题 | 每次改模型后执行 `dotnet ef migrations add` 并测试 |
| AI 生成的代码不稳定或逻辑跑不通 | 功能不可用 | AI 代码必须人工走读 + 手动测试，不能直接信任 |
| 学生没有 .NET 9 SDK | 项目无法启动 | 统一要求安装 .NET 9 SDK，课上检查环境 |
| 界面太丑影响演示印象 | 呈现效果差 | 使用 Bootstrap 自带组件 + 模板，不花时间在自定义 CSS 上 |
