## 4. 运行时问题修复

### 4.1 Reservations.cshtml ViewBag 类型转换

| 项目 | 内容 |
|------|------|
| **问题** | `ViewBag.Reservations as List<dynamic>` 因泛型协变限制返回 null → 页面始终显示空状态 |
| **原因** | `List<匿名类型>` 不可转换为 `List<dynamic>`（C# 泛型不变性） |
| **修复** | 改为 `((IEnumerable)ViewBag.Reservations)?.Cast<dynamic>().ToList()` |
| **文件** | `Views/Admin/Reservations.cshtml` |
| **验证** | ✅ 修复后 6 条预约记录正确显示 |

### 4.2 StatusHelper.GetReservationDisplayStatus 缺少日期参数

| 项目 | 内容 |
|------|------|
| **问题** | 动态状态计算只检查 `TimeOfDay` 不检查 `Date`，导致未来日期的预约显示错误状态（如明天 09:00-11:00 显示"已完成"） |
| **修复** | 增加 `DateOnly date` 参数：未来日期 → "待开始"，过去日期 → "已完成"，今日 → 时间判断 |
| **文件** | `Helpers/StatusHelper.cs`、`Controllers/AdminController.cs`、`Controllers/ReservationController.cs`、`Controllers/SeatsController.cs` |
| **验证** | ✅ 2026-07-14 的 B-01 正确显示"待开始"而非"已完成" |

### 4.3 CanCancel 缺少日期判断

| 项目 | 内容 |
|------|------|
| **问题** | `CanCancel = Status == "Pending" && TimeOfDay < StartTime` 未考虑 Date，未来日期的预约即使按钮可见，POST 时也会被错误拦截 |
| **修复** | 改为 `(Date > today) || (Date == today && TimeOfDay < StartTime)` |
| **文件** | `Controllers/AdminController.cs`（CanCancelAdmin）、`Controllers/ReservationController.cs`（CanCancel） |
| **验证** | ✅ 未来日期的预约按钮正常显示，POST 操作正常 |
