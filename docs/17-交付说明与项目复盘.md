## 3. 做得好的地方

### 3.1 工程规范

- **严格的分层架构**：Controllers / Models / Data / Helpers / Filters 各司其职，职责单一
- **文档驱动开发**：从立项到交付，17 份文档与代码保持同步，每轮有审计记录
- **跨文档一致性审计**：开发前执行了完整的 11 项一致性检查，修正了 5 处文档-代码偏差
- **自动化测试体系建设**：Playwright 端到端测试 + Node.js 脚本测试 + 兼容性测试，三层验证

### 3.2 功能设计

- **轻量账号体系**：体验账号切换，无需注册登录，降低使用门槛
- **动态状态计算**：StatusHelper 根据持久化状态 + 实时时间计算展示状态，无需复杂的状态机
- **多层校验守卫**：预约提交有表单校验 + 冲突检测 + 状态守卫 + Session 守卫，防止异常数据
- **日期感知取消**：仅允许取消未开始的预约，已开始/已结束的预约自动禁用取消按钮

### 3.3 缺陷管理

- **完整的 Bug 闭环**：从 Sprint 1 到 Sprint 4，所有发现的 Bug 均记录、修复、验证关闭
- **P0/P1 清零**：所有阻塞性、严重性问题均已修复

## 6. 最终演示顺序建议

以下为向老师或同学演示本项目的推荐顺序，总时长约 **5-8 分钟**：

### Step 1：首页与账号切换（1 分钟）
1. 访问 `http://localhost:5211`
2. 展示首页标题"图书馆座位预约系统"
3. 从下拉框选择"学生A"，观察导航栏显示"学生A"
4. 再次切换为"学生B"，观察 Session 持久化

### Step 2：座位浏览与筛选（1 分钟）
1. 点击导航栏"座位列表"
2. 展示 12 座位网格，观察各座位状态（空闲/已预约/使用中）
3. 选择区域筛选"二楼阅览室"，观察仅显示 B 区座位
4. 点击某个座位，进入座位详情页
5. 展示当日预约时段列表

### Step 3：预约流程（1.5 分钟）
1. 在座位详情页点击"预约此座位"
2. 填写时段（如 10:00-11:00）
3. 提交预约
4. 观察成功提示"预约成功！"
5. 自动跳转到"我的预约"页面

### Step 4：取消预约（1 分钟）
1. 在我的预约页面，找到刚创建的预约
2. 点击"取消"，确认弹窗
3. 观察成功提示"预约已取消"，状态变为"已取消"

### Step 5：管理端操作（2 分钟）
1. 点击导航栏"管理后台"（或直接访问 `/Admin/Login`）
2. 输入 admin / admin123 登录
3. 展示预约管理页，按状态筛选"待开始"
4. 进入"座位管理"，展示 12 座位列表
5. 将某个座位设为"维护中"，观察状态变化
6. 进入"统计页"，展示座位总数/今日预约/使用率/区域分布

### Step 6：自动化测试展示（1 分钟）
1. 运行 `node scripts/smoke-test.mjs`
2. 展示 10/10 全部通过
3. （可选）运行 `npx playwright test --reporter=list`
4. 展示 9/9 全部通过
