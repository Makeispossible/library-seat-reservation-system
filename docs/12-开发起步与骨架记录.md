## 3. 数据库表与结构

### 3.1 EF Core 迁移信息

| 项 | 内容 |
|---|------|
| 迁移名称 | `InitialCreate` |
| 迁移时间戳 | `20260713025059` |
| 建库方式 | Code First + `dotnet ef database update` |
| 数据库名 | `LibrarySeatReservation` |
| 数据库位置 | SQL Server LocalDB `(localdb)\MSSQLLocalDB` |
| 种子数据写入时机 | 应用启动时 `Program.cs` 中 `DbInitializer.Initialize()` |

### 3.2 三张表结构

#### StudentUsers

| 列名 | 类型 | 约束 | 备注 |
|------|------|------|------|
| Id | INT | PK, IDENTITY | 主键 |
| Name | NVARCHAR(50) | NOT NULL | 体验账号名 |

种子数据：学生A、学生B、学生C、学生D、学生E

#### Seats

| 列名 | 类型 | 约束 | 备注 |
|------|------|------|------|
| Id | INT | PK, IDENTITY | 主键 |
| SeatNumber | NVARCHAR(20) | NOT NULL | 座位编号，如 A-01 |
| Area | NVARCHAR(50) | NOT NULL | 区域名 |
| Floor | NVARCHAR(10) | NOT NULL | 楼层 |
| Status | NVARCHAR(20) | NOT NULL, DEFAULT 'Available' | Available / Maintenance |

种子数据：12 条（A-01 ~ C-04，3 区域 × 4 座位）

索引：`IX_Seats_Area_Floor`（Area, Floor）

#### Reservations

| 列名 | 类型 | 约束 | 备注 |
|------|------|------|------|
| Id | INT | PK, IDENTITY | 主键 |
| SeatId | INT | FK → Seats.Id, CASCADE | 外键 |
| StudentUserId | INT | FK → StudentUsers.Id, CASCADE | 外键 |
| Date | DATE | NOT NULL | 预约日期 |
| StartTime | TIME | NOT NULL | 开始时间 |
| EndTime | TIME | NOT NULL | 结束时间 |
| Status | NVARCHAR(20) | NOT NULL, DEFAULT 'Pending' | Pending / Cancelled |
| CreatedAt | DATETIME2 | NOT NULL, DEFAULT GETDATE() | 创建时间 |

索引：
- `IX_Reservations_SeatId_Date`（SeatId, Date）
- `IX_Reservations_StudentUserId`（StudentUserId）
- `IX_Reservations_Date_StartTime`（Date, StartTime）

## 4. 关键决策记录

### 4.1 编码阶段决策

| 决策项 | 选择 | 替代方案 | 选择原因 |
|--------|------|---------|---------|
| 建表方式 | EF Core Migrations | EnsureCreated | EnsureCreated 不能走迁移流水线，不适合后续 Schema 变更 |
| Migration 触发时机 | `Program.cs` 中 `db.Database.Migrate()` | 手动 `dotnet ef database update` | 自动执行挂起迁移，保证首次运行即建库 |
| Date 类型 | `DateOnly` | `DateTime` | .NET 6+ 引入 DateOnly，语义更精确，避免时间部分干扰 |
| 验证方式 | 服务端 ModelState + 显式检查 | 无 | 所有验证在 POST Action 中显式执行 |
| 删除页面 | 模板默认 Privacy 等全部删除 | 保留 | 本项目无这些页面，减少混淆 |
| Bootstrap | CDN 5.3.3 | npm/local | 零配置即可运行 |
| 管理端 Layout | 独立 `_AdminLayout.cshtml` | 共用 `_Layout.cshtml` | 管理端深色导航栏与用户端蓝色完全不同，独立 Layout 更清晰 |
