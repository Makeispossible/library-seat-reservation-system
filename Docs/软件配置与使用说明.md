# 软件配置与使用说明

## 运行环境

- Windows 10/11（推荐）、Linux 或 macOS
- .NET SDK 9.0 或更高版本
- SQL Server LocalDB（Windows 自带）或 SQLite（降级方案）
- Edge、Chrome 或 Firefox 浏览器
- Node.js ≥ 18（可选，运行自动化测试用）

## 安装与配置

### 1. 确认依赖

```bash
dotnet --version    # 应 ≥ 9.0
SqlLocalDB info     # 应显示可用实例
dotnet restore      # 还原 NuGet 包
```

### 2. 首次运行

```bash
dotnet run
```

浏览器访问 http://localhost:5211。

首次启动会自动创建数据库、建表、写入种子数据（5 个体验学生 + 12 个座位）。

### 3. 运行自动化测试

```powershell
# 安装 Playwright 依赖（仅首次）
npm install

# Playwright E2E 测试
npx playwright test --reporter=list

# Node.js 烟雾测试
node scripts/smoke-test.mjs
```

## 使用指南

### 学生端

1. 访问首页 http://localhost:5211
2. 从顶部导航栏下拉框选择体验账号（学生A ~ 学生E）
3. 点击"座位列表"浏览所有座位，可用区域/楼层筛选
4. 点击座位卡片进入详情页，查看今日预约时段
5. 点击"预约此座位"，填写时段提交预约
6. 进入"我的预约"查看记录，可取消未开始的预约

### 管理端

1. 点击导航栏"管理后台"或直接访问 http://localhost:5211/Admin/Login
2. 使用 admin / admin123 登录
3. 预约管理：查看全部预约，按日期/状态/用户筛选，取消预约
4. 座位管理：查看座位列表，添加新座位，切换可用/维护状态
5. 统计页：查看座位总数、今日预约数、使用率、区域分布

## 常见问题

**Q: 启动报错提示 LocalDB 不可用？**
A: 可使用 SQLite 降级方案，修改 Program.cs 中的 UseSqlServer 为 UseSqlite。

**Q: 如何重新生成种子数据？**
A: 执行 `dotnet ef database drop` 后重新 `dotnet run`。

**Q: Playwright 测试运行失败？**
A: 确认应用已启动（http://localhost:5211 可访问），且已执行 `npm install`。
